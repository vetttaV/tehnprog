Техническая спецификация программного комплекса

1. Общая структурная схема и взаимодействие модулей

Программный комплекс построен на основе многоуровневой архитектуры. Взаимодействие компонентов организовано в виде последовательной передачи данных от графического интерфейса пользователя к вычислительному ядру, с обеспечением строгой типизации и инкапсуляции на каждом уровне.

[ Уровень 1: Пользовательский интерфейс (PyQt5) ] | Сбор данных от пользователя, формирование запросов v [ Уровень 2: Интеграционный слой (Python) ] | Загрузка Dylib, подготовка типов данных (ctypes) v [ Уровень 3: Слой экспорта (C-Wrapper) ] | Предоставление C-API v [ Уровень 4: Вычислительное ядро (C++) ] | Объектная модель сообщений, контроль интенсивности (Rate Limiting) [ База данных в оперативной памяти (std::vector, std::map) ]

Логика взаимодействия: Пользователь вводит параметры сетевого пакета через графический интерфейс. Эти данные передаются в класс-обертку на Python, который конвертирует строки в массивы символов и вызывает бинарные функции интерфейса. Функции экспорта создают объекты C++ и вызывают методы контроля лимитов в ядре. В случае успешного прохождения проверок сообщение сохраняется во внутренней структуре данных.

2. Система сборки (CMake)

Процесс компиляции ядра и слоя экспорта управляется кроссплатформенной системой сборки CMake.

CMakeLists.txt

Файл конфигурации выполняет следующие задачи:

  • Устанавливает минимальную требуемую версию CMake (3.10) и инициализирует проект unicastmessage_lib.
  • Определяет стандарт языка C++11 (CMAKE_CXX_STANDARD 11).
  • Объединяет исходные файлы (unicastmessage.cpp, wrapper.cpp) и компилирует их в общую динамическую библиотеку (SHARED), готовую к загрузке интерпретатором Python.
  • Настраивает директории заголовочных файлов для корректной линковки.

3. Описание компонентов ядра (C++ Core)

Ядро включает в себя базовые классы для представления сетевых пакетов и контроллер для управления ими.

C++NetworkMessage(double time, uint32_t sender, uint32_t len, string protocol)

Конструктор. Инициализирует основные атрибуты пакета.

ПараметрТипНазначение

senderuint32_tIP-адрес отправителя в числовом виде.
lenuint32_tРазмер полезной нагрузки в байтах.

C++virtual bool isValid() const = 0

Абстрактный метод для проверки логической корректности пакета.

Возвращает: bool (true — данные валидны, false — ошибка).
C++Геттеры и Сеттеры

Методы getRegistrationTime(), getSenderIP(), getLength(), getProtocolName() возвращают соответствующие параметры. Сеттеры позволяют изменять эти поля.

Возвращают: Значения соответствующих внутренних переменных.
C++static uint32_t ipStringToUint(const string& ipStr)

Утилитарный метод для преобразования строкового IP в число.

ПараметрТипНазначение

2.2. Классы-наследники сообщений

  • UnicastMessage: Реализует метод isValid(). Проверяет, что IP отправителя (senderIP) не равен IP получателя (receiverIP).
  • BroadcastMessage: Содержит маску подсети (subnetMask). Метод isValid() проверяет корректность маски (отсутствие чередования нулей и единиц).
  • MulticastMessage: Содержит идентификатор группы (groupID). Метод isValid() проверяет вхождение IP в мультикаст-диапазон (224.0.0.0 — 239.255.255.255).

Главный контроллер, управляющий регистрацией пакетов и проверкой интенсивности.

Реализует алгоритм скользящего окна. Удаляет метки времени, выходящие за пределы окна, и проверяет количество оставшихся.

ПараметрТипНазначение
senderIPuint32_tIP-адрес отправителя для проверки.
currentTimedoubleТекущее системное время.
Возвращает: bool (true — лимит не превышен, false — превышен).
C++bool addUnicast(double time, const string& sender, const string& receiver, uint32_t length, const string& protocol)

Создает пакет, проверяет его валидность и пропускает через фильтр лимитов. Аналогично работают addBroadcast и addMulticast.

ПараметрТипНазначение
time, lengthdouble, uint32_tВремя и размер.
sender, receiver, protocolstringСтроковые параметры пакета.
Возвращает: bool (true — успешно добавлено, false — отклонено).
C++void removeOldMessages(double olderThanTime)

Выполняет очистку оперативной памяти от сообщений, зарегистрированных ранее указанного времени.

ПараметрТипНазначение
olderThanTimedoubleПорог времени для удаления.
Возвращает: void.

Конвертирует Python-строки в байтовые массивы (ctypes) и вызывает C-функцию.
